<Note>
  This quickstart will guide you through cloning the repository, opening it in Android Studio, and running the app on an emulator or physical device.
</Note>

## Prerequisites

Before you begin, ensure you have the following installed:

- **Android Studio** Ladybug | 2024.2.1 or later
- **JDK 11** or higher
- **Android SDK** with the following:
  - Minimum SDK: API 28 (Android 9.0)
  - Target SDK: API 36 (Android 15)
  - Compile SDK: API 36
- **Gradle 9.2.1** (included via wrapper)

<Tip>
  The project uses Gradle wrapper, so you don't need to install Gradle manually. The correct version will be downloaded automatically.
</Tip>

## Quick Start

<Steps>
  <Step title="Clone the Repository">
    Clone the VetyCare repository to your local machine:

    <CodeGroup>
      ```bash HTTPS
      git clone https://github.com/your-org/vetycare.git
      cd vetycare
      ```

      ```bash SSH
      git clone git@github.com:your-org/vetycare.git
      cd vetycare
      ```
    </CodeGroup>
  </Step>

  <Step title="Open in Android Studio">
    1. Launch Android Studio
    2. Click **File** → **Open**
    3. Navigate to the cloned `vetycare` directory
    4. Select the root folder and click **OK**
    5. Wait for Gradle to sync (this may take a few minutes on first launch)

    <Note>
      Android Studio will automatically download Gradle 9.2.1 and all required dependencies during the initial sync.
    </Note>
  </Step>

  <Step title="Configure an Android Virtual Device (AVD)">
    If you don't have an emulator set up:

    1. Click the **Device Manager** icon in the toolbar
    2. Click **Create Device**
    3. Select a device definition (e.g., Pixel 6)
    4. Select a system image with API 28 or higher (API 36 recommended)
    5. Click **Finish**

    <Tip>
      For the best experience, select an API 36 (Android 15) system image to match the target SDK.
    </Tip>
  </Step>

  <Step title="Run the Application">
    There are two ways to run the app:

    **Option 1: Using Android Studio**
    1. Select your device/emulator from the device dropdown
    2. Click the green **Run** button (▶️) or press `Shift + F10`
    3. The app will build and launch automatically

    **Option 2: Using Gradle Command Line**
    <CodeGroup>
      ```bash Linux/macOS
      ./gradlew installDebug
      ```

      ```bash Windows
      gradlew.bat installDebug
      ```
    </CodeGroup>
  </Step>
</Steps>

## Understanding the App

VetyCare is built with:

- **Kotlin** as the primary language
- **Material Design 3** (version 1.13.0) for modern UI components
- **View Binding** for type-safe view access
- **Navigation Component** (version 2.9.7) for fragment navigation
- **AndroidX libraries** for backward compatibility

### Main Components

The app launches with `MainActivity`, which sets up:

```kotlin MainActivity.kt:19-36
override fun onCreate(savedInstanceState: Bundle?) {
    super.onCreate(savedInstanceState)

    binding = ActivityMainBinding.inflate(layoutInflater)
    setContentView(binding.root)

    setSupportActionBar(binding.toolbar)

    val navController = findNavController(R.id.nav_host_fragment_content_main)
    appBarConfiguration = AppBarConfiguration(navController.graph)
    setupActionBarWithNavController(navController, appBarConfiguration)

    binding.fab.setOnClickListener { view ->
        Snackbar.make(view, "Replace with your own action", Snackbar.LENGTH_LONG)
            .setAction("Action", null)
            .setAnchorView(R.id.fab).show()
    }
}
```

Key features:
- Navigation with back stack support
- Floating Action Button (FAB) with Snackbar interactions
- Material toolbar with app bar configuration
- Type-safe view binding

## Verify Installation

Once the app launches, you should see:

1. A Material Design 3 toolbar at the top
2. The first fragment content area
3. A Floating Action Button (FAB) in the bottom-right corner
4. Clicking the FAB displays a Snackbar message
5. Navigation between fragments using the UI buttons

<Warning>
  If the build fails, ensure you have the Android SDK Platform 36 installed via SDK Manager.
</Warning>
