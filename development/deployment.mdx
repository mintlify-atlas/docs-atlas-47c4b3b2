## Overview

Deploying VetyCare to production involves generating a signed release build, configuring app versioning, and preparing the app for distribution through the Google Play Store.

## Prerequisites

Before deploying to production, ensure you have:

- **Completed testing** - All unit and instrumented tests passing
- **Release build configuration** - Properly configured in `build.gradle.kts`
- **Google Play Console account** - For app distribution
- **Signing key** - For app authentication

## Version Management

App versioning is critical for updates and user communication.

### Current Version Configuration

Versions are defined in `app/build.gradle.kts`:

```gradle
defaultConfig {
    applicationId = "com.example.vetycare"
    minSdk = 28
    targetSdk = 36
    versionCode = 1
    versionName = "1.0"
}
```

### Version Numbers Explained

<CardGroup cols={2}>
  <Card title="versionCode" icon="hashtag">
    **Current: 1**
    
    Internal version number used by Google Play to manage updates. Must be incremented with each release.
  </Card>
  <Card title="versionName" icon="tag">
    **Current: 1.0**
    
    User-facing version string displayed in the Play Store and app settings.
  </Card>
</CardGroup>

### Updating Versions

<Steps>
  <Step title="Increment versionCode">
    Always increment `versionCode` before releasing:
    
    ```gradle
    versionCode = 2  // Increment by 1
    ```
  </Step>

  <Step title="Update versionName">
    Update the user-facing version following semantic versioning:
    
    ```gradle
    versionName = "1.1"      // Minor update
    versionName = "1.0.1"    // Patch update
    versionName = "2.0"      // Major update
    ```
  </Step>
</Steps>

<Warning>
Google Play will reject uploads with a `versionCode` less than or equal to the current production version.
</Warning>

## Release Build Configuration

The release build type is configured in `app/build.gradle.kts`:

```gradle
buildTypes {
    release {
        isMinifyEnabled = false
        proguardFiles(
            getDefaultProguardFile("proguard-android-optimize.txt"),
            "proguard-rules.pro"
        )
    }
}
```

### Recommended Production Settings

For production releases, enable code minification and optimization:

```gradle
buildTypes {
    release {
        isMinifyEnabled = true        // Enable code shrinking
        isShrinkResources = true      // Remove unused resources
        proguardFiles(
            getDefaultProguardFile("proguard-android-optimize.txt"),
            "proguard-rules.pro"
        )
    }
}
```

<Note>
After enabling minification, thoroughly test the release build to ensure ProGuard doesn't break functionality.
</Note>

## Generating a Signing Key

Android requires all apps to be digitally signed before installation.

<Steps>
  <Step title="Generate Keystore">
    Use the Java `keytool` to create a new keystore:
    
    ```bash
    keytool -genkey -v -keystore vetycare-release.keystore \
      -alias vetycare \
      -keyalg RSA \
      -keysize 2048 \
      -validity 10000
    ```
    
    You'll be prompted for:
    - **Keystore password** - Store securely
    - **Key password** - Store securely
    - **Organization details** - Your information
  </Step>

  <Step title="Store Keystore Securely">
    Move the keystore to a secure location:
    
    ```bash
    mkdir -p ~/.android/keystores
    mv vetycare-release.keystore ~/.android/keystores/
    ```
    
    <Warning>
    Never commit your keystore to version control. Add it to `.gitignore`.
    </Warning>
  </Step>

  <Step title="Create keystore.properties">
    Create a file `keystore.properties` in your project root:
    
    ```properties
    storePassword=YourStorePassword
    keyPassword=YourKeyPassword
    keyAlias=vetycare
    storeFile=/home/user/.android/keystores/vetycare-release.keystore
    ```
    
    Add to `.gitignore`:
    ```
    keystore.properties
    *.keystore
    *.jks
    ```
  </Step>
</Steps>

<Warning>
**Critical**: If you lose your signing key, you cannot update your app on Google Play. Store it securely with backups.
</Warning>

## Configuring Signing in Gradle

Update `app/build.gradle.kts` to use your signing key:

<Steps>
  <Step title="Load Keystore Properties">
    Add at the top of `app/build.gradle.kts`:
    
    ```kotlin
    import java.util.Properties
    import java.io.FileInputStream
    
    val keystorePropertiesFile = rootProject.file("keystore.properties")
    val keystoreProperties = Properties()
    if (keystorePropertiesFile.exists()) {
        keystoreProperties.load(FileInputStream(keystorePropertiesFile))
    }
    ```
  </Step>

  <Step title="Configure Signing Config">
    Add signing configuration inside the `android` block:
    
    ```kotlin
    android {
        signingConfigs {
            create("release") {
                if (keystorePropertiesFile.exists()) {
                    storeFile = file(keystoreProperties["storeFile"] as String)
                    storePassword = keystoreProperties["storePassword"] as String
                    keyAlias = keystoreProperties["keyAlias"] as String
                    keyPassword = keystoreProperties["keyPassword"] as String
                }
            }
        }
        
        buildTypes {
            release {
                signingConfig = signingConfigs.getByName("release")
                isMinifyEnabled = true
                isShrinkResources = true
                proguardFiles(
                    getDefaultProguardFile("proguard-android-optimize.txt"),
                    "proguard-rules.pro"
                )
            }
        }
    }
    ```
  </Step>
</Steps>

## Building the Release APK

With signing configured, build the release APK:

<Steps>
  <Step title="Clean Previous Builds">
    ```bash
    ./gradlew clean
    ```
  </Step>

  <Step title="Build Release APK">
    ```bash
    ./gradlew assembleRelease
    ```
    
    The signed APK will be at:
    ```
    app/build/outputs/apk/release/app-release.apk
    ```
  </Step>

  <Step title="Verify Signing">
    Verify the APK is properly signed:
    
    ```bash
    jarsigner -verify -verbose -certs app/build/outputs/apk/release/app-release.apk
    ```
    
    Look for "jar verified" in the output.
  </Step>
</Steps>

## Building Android App Bundle (Recommended)

Google Play recommends using Android App Bundles (AAB) instead of APKs:

<Steps>
  <Step title="Build Release Bundle">
    ```bash
    ./gradlew bundleRelease
    ```
    
    The signed bundle will be at:
    ```
    app/build/outputs/bundle/release/app-release.aab
    ```
  </Step>

  <Step title="Benefits of AAB">
    - **Smaller downloads** - Google Play generates optimized APKs per device
    - **Dynamic delivery** - Install only necessary features
    - **Required for new apps** - Mandatory for new apps since August 2021
  </Step>
</Steps>

<Tip>
Use AAB for Play Store distribution and APK for testing or alternative distribution.
</Tip>

## Backup and Data Extraction Rules

VetyCare includes configurations for Android's backup and data extraction features.

### Backup Rules

Defined in `app/src/main/res/xml/backup_rules.xml`:

```xml
<?xml version="1.0" encoding="utf-8"?>
<full-backup-content>
    <!--
    <include domain="sharedpref" path="."/>
    <exclude domain="sharedpref" path="device.xml"/>
    -->
</full-backup-content>
```

### Data Extraction Rules

Defined in `app/src/main/res/xml/data_extraction_rules.xml`:

```xml
<?xml version="1.0" encoding="utf-8"?>
<data-extraction-rules>
    <cloud-backup>
        <!-- TODO: Use <include> and <exclude> to control what is backed up.
        <include .../>
        <exclude .../>
        -->
    </cloud-backup>
</data-extraction-rules>
```

<Note>
Customize these files to control which app data is backed up to Google Drive and included in device-to-device transfers.
</Note>

## Uploading to Google Play Console

<Steps>
  <Step title="Access Play Console">
    Log in to [Google Play Console](https://play.google.com/console)
  </Step>

  <Step title="Create or Select App">
    Create a new app or select your existing VetyCare app
  </Step>

  <Step title="Upload Bundle">
    Navigate to **Production** > **Create new release**
    
    Upload your `app-release.aab` file
  </Step>

  <Step title="Complete Release Details">
    - Add release notes
    - Review app content
    - Submit for review
  </Step>
</Steps>

## Troubleshooting

### Signing Issues

**Error: "Keystore was tampered with, or password was incorrect"**

- Verify password in `keystore.properties`
- Ensure keystore file path is correct
- Check file permissions

### ProGuard Issues

**App crashes in release but works in debug:**

- Add necessary keep rules to `proguard-rules.pro`
- Check ProGuard output for removed classes
- Test release build thoroughly before deployment

### Version Conflicts

**Play Console rejects upload:**

- Ensure `versionCode` is greater than current production version
- Verify `targetSdk` meets current requirements (currently 36 for new apps)
