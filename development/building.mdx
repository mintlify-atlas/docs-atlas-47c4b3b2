## Overview

VetyCare uses Gradle as its build system. The app supports multiple build types optimized for different stages of development and distribution.

## Prerequisites

Before building the app, ensure you have:

- **JDK 11 or higher** - Required for compiling the app
- **Android SDK** - With API level 36 (Android 15)
- **Gradle Wrapper** - Included in the project (no separate installation needed)

## Build Configuration

The app's build configuration is defined in `app/build.gradle.kts`:

<CodeGroup>
```gradle app/build.gradle.kts
android {
    namespace = "com.example.vetycare"
    compileSdk {
        version = release(36) {
            minorApiLevel = 1
        }
    }

    defaultConfig {
        applicationId = "com.example.vetycare"
        minSdk = 28
        targetSdk = 36
        versionCode = 1
        versionName = "1.0"
    }

    compileOptions {
        sourceCompatibility = JavaVersion.VERSION_11
        targetCompatibility = JavaVersion.VERSION_11
    }

    buildFeatures {
        viewBinding = true
    }
}
```
</CodeGroup>

## Build Types

VetyCare supports two build types:

### Debug Build

Optimized for development with debugging enabled:

- **Minification**: Disabled
- **Debugging**: Enabled
- **ProGuard**: Not applied
- **Signing**: Uses debug keystore automatically

### Release Build

Optimized for production distribution:

```gradle
buildTypes {
    release {
        isMinifyEnabled = false
        proguardFiles(
            getDefaultProguardFile("proguard-android-optimize.txt"),
            "proguard-rules.pro"
        )
    }
}
```

<Note>
Currently, code minification is disabled (`isMinifyEnabled = false`). Enable it before releasing to production to reduce APK size and obfuscate code.
</Note>

## Building with Gradle

### Using the Gradle Wrapper

The project includes Gradle Wrapper scripts that ensure everyone uses the same Gradle version:

- **Linux/Mac**: `./gradlew`
- **Windows**: `gradlew.bat`

<Tip>
Always use the Gradle Wrapper instead of a globally installed Gradle to ensure consistent builds across different environments.
</Tip>

### Build Commands

<Steps>
  <Step title="Navigate to Project Root">
    Open a terminal and navigate to the VetyCare project directory:
    
    ```bash
    cd /path/to/vetycare
    ```
  </Step>

  <Step title="Build Debug APK">
    Build a debug version for testing:
    
    ```bash
    ./gradlew assembleDebug
    ```
    
    This creates an APK at:
    ```
    app/build/outputs/apk/debug/app-debug.apk
    ```
  </Step>

  <Step title="Build Release APK">
    Build a release version for distribution:
    
    ```bash
    ./gradlew assembleRelease
    ```
    
    This creates an APK at:
    ```
    app/build/outputs/apk/release/app-release.apk
    ```
  </Step>
</Steps>

### Additional Build Commands

<CodeGroup>
```bash Clean Build
# Remove all build artifacts
./gradlew clean
```

```bash Clean + Build
# Clean and build in one command
./gradlew clean assembleDebug
```

```bash Build All Variants
# Build all build types
./gradlew assemble
```

```bash Build Bundle
# Build Android App Bundle for Play Store
./gradlew bundleRelease
```
</CodeGroup>

ProGuard rules are defined in `app/proguard-rules.pro`. The default configuration includes:

```pro

#   public *;

# Uncomment this to preserve the line number information for

# If you keep the line number information, uncomment this to
```

<Warning>
Before enabling minification for release builds, thoroughly test the app to ensure ProGuard doesn't remove necessary classes.
</Warning>

## Build Features

### View Binding

View Binding is enabled in the project:

```gradle
buildFeatures {
    viewBinding = true
}
```

This generates binding classes for each XML layout file, providing type-safe access to views.

## Gradle Properties

Project-wide settings are configured in `gradle.properties`:

```properties
# JVM memory settings
org.gradle.jvmargs=-Xmx2048m -Dfile.encoding=UTF-8

# AndroidX support
android.useAndroidX=true

# Kotlin code style
kotlin.code.style=official

# R class optimization
android.nonTransitiveRClass=true
```

## Troubleshooting

### Build Fails with Memory Error

Increase heap size in `gradle.properties`:

```properties
org.gradle.jvmargs=-Xmx4096m -Dfile.encoding=UTF-8
```

### Gradle Daemon Issues

Stop all Gradle daemons:

```bash
./gradlew --stop
```

### Dependency Resolution Issues

Sync and refresh dependencies:

```bash
./gradlew --refresh-dependencies
```
