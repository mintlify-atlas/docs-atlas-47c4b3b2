VetyCare follows the standard Android project structure with organized separation between source code, resources, and build configuration. This page documents the complete project structure and explains the purpose of each directory.

## Project Overview

```
VetyCare/
├── app/                          # Main application module
│   ├── src/
│   │   ├── main/                 # Main source set
│   │   ├── androidTest/          # Instrumentation tests
│   │   └── test/                 # Unit tests
│   ├── build.gradle.kts          # App module build configuration
│   └── proguard-rules.pro        # ProGuard rules for code obfuscation
├── gradle/                       # Gradle wrapper files
├── build.gradle.kts              # Project-level build configuration
├── settings.gradle.kts           # Project settings and module inclusion
├── gradle.properties             # Gradle configuration properties
└── gradlew / gradlew.bat         # Gradle wrapper scripts
```

## Resources Directory Structure

The `app/src/main/res/` directory contains all application resources:

### Resource Categories

<AccordionGroup>
  <Accordion title="Layout Resources (res/layout/)">
    XML files defining the UI structure for activities and fragments.
    
    ```
    res/layout/
    ├── activity_main.xml          # Main activity layout
    ├── content_main.xml           # Content container for navigation
    ├── fragment_first.xml         # First fragment layout
    └── fragment_second.xml        # Second fragment layout
    ```
    
    **activity_main.xml** structure:
    ```xml
    <androidx.coordinatorlayout.widget.CoordinatorLayout>
        <com.google.android.material.appbar.AppBarLayout>
            <com.google.android.material.appbar.MaterialToolbar />
        </com.google.android.material.appbar.AppBarLayout>
        
        <include layout="@layout/content_main" />
        
        <com.google.android.material.floatingactionbutton.FloatingActionButton />
    </androidx.coordinatorlayout.widget.CoordinatorLayout>
    ```
  </Accordion>
  
  <Accordion title="Navigation Resources (res/navigation/)">
    Navigation graph defining app navigation structure.
    
    ```
    res/navigation/
    └── nav_graph.xml              # Navigation graph definition
    ```
    
    See the [Navigation](/architecture/navigation) page for detailed information.
  </Accordion>
  
  <Accordion title="Drawable Resources (res/drawable/)">
    Vector graphics and drawable resources.
    
    ```
    res/drawable/
    ├── ic_launcher_background.xml  # Launcher icon background
    └── ic_launcher_foreground.xml  # Launcher icon foreground
    ```
  </Accordion>
  
  <Accordion title="Menu Resources (res/menu/)">
    Menu definitions for the action bar.
    
    ```
    res/menu/
    └── menu_main.xml              # Main activity menu
    ```
  </Accordion>
  
  <Accordion title="Mipmap Resources (res/mipmap-*)">
    App launcher icons at different densities.
    
    ```
    res/mipmap-anydpi/             # Adaptive icons
    res/mipmap-mdpi/               # Medium density icons
    res/mipmap-hdpi/               # High density icons

    res/mipmap-xxhdpi/             # Extra-extra-high density icons
    res/mipmap-xxxhdpi/            # Extra-extra-extra-high density icons
    ```
  </Accordion>
  
  <Accordion title="Values Resources (res/values/)">
    String, color, dimension, and theme definitions.
    
    ```
    res/values/
    ├── strings.xml                # String resources
    ├── colors.xml                 # Color definitions
    ├── dimens.xml                 # Dimension values
    └── themes.xml                 # Material 3 themes
    
    res/values-night/
    └── themes.xml                 # Dark theme overrides
    
    res/values-v23/
    └── themes.xml                 # API 23+ theme overrides
    
    res/values-land/
    └── dimens.xml                 # Landscape dimensions
    
    res/values-w600dp/
    └── dimens.xml                 # 600dp+ width dimensions
    
    res/values-w1240dp/
    └── dimens.xml                 # 1240dp+ width dimensions
    ```
  </Accordion>
  
  <Accordion title="XML Resources (res/xml/)">
    XML configuration files.
    
    ```
    res/xml/
    ├── backup_rules.xml           # Backup configuration
    └── data_extraction_rules.xml  # Data extraction rules
    ```
  </Accordion>
</AccordionGroup>

### Settings Configuration

Location: `settings.gradle.kts`

```gradle
pluginManagement {
    repositories {
        google {
            content {
                includeGroupByRegex("com\\.android.*")
                includeGroupByRegex("com\\.google.*")
                includeGroupByRegex("androidx.*")
            }
        }
        mavenCentral()
        gradlePluginPortal()
    }
}

dependencyResolutionManagement {
    repositoriesMode.set(RepositoriesMode.FAIL_ON_PROJECT_REPOS)
    repositories {
        google()
        mavenCentral()
    }
}

rootProject.name = "VetyCare"
include(":app")
```

<Note>
  The project uses **centralized repository management** with `FAIL_ON_PROJECT_REPOS` mode to ensure all repositories are declared in settings.gradle.kts rather than in individual module build files.
</Note>

## Gradle Wrapper

The project includes Gradle wrapper scripts for consistent builds:

- **gradlew** (Linux/Mac): `./gradlew <task>`
- **gradlew.bat** (Windows): `gradlew.bat <task>`
- **gradle/wrapper/**: Wrapper JAR and properties

### Common Gradle Commands

```bash
# Build the app
./gradlew build

# Install debug APK on connected device
./gradlew installDebug

# Run unit tests
./gradlew test

# Run instrumentation tests
./gradlew connectedAndroidTest

# Clean build artifacts
./gradlew clean
```

## Project Structure Best Practices

<Steps>
  <Step title="Package Organization">
    Keep all related source files in the `com.example.vetycare` package. As the app grows, create subpackages for features:
    ```
    com.example.vetycare/
    ├── ui/
    │   ├── home/
    │   └── profile/
    ├── data/
    │   ├── model/
    │   └── repository/
    └── util/
    ```
  </Step>
  
  <Step title="Resource Naming">
    Follow Android naming conventions:
    - Layouts: `activity_main.xml`, `fragment_first.xml`
    - IDs: `button_submit`, `text_title`
    - Drawables: `ic_launcher_foreground.xml`
    - Colors: `color_primary`, `color_secondary`
  </Step>
  
  <Step title="Configuration Management">
    Keep configuration in appropriate places:
    - App-specific config: `app/build.gradle.kts`
    - Project-wide config: `build.gradle.kts`
    - Gradle properties: `gradle.properties`
  </Step>
</Steps>
